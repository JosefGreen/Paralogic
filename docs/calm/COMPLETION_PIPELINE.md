# CALM / Paper Armies Completion Pipeline

## Purpose

This pipeline governs completion of the full CALM doctrine and Paper Armies
book ecosystem. It is broader than the Paralogic formal workbench. Paralogic
concepts may inform audit discipline, but the repository does not prove CALM
and does not validate defense claims.

## Current Execution Slice

This pass executes the first part of Phase 0:

1. Establishes the project charter.
2. Adds a Paper Armies book architecture.
3. Adds a source and case dossier template.
4. Adds a Lean-checked CALM guard core for claim travel, learning travel, and
   Paper CALM reflexivity.
5. Adds a static checker so the new workstream has a verification hook.

## Gate Rules

No phase may be marked complete unless the artifact exists, is linked from this
pipeline, and has an explicit claim boundary.

No case may be used as evidence until it has a source maturity rating,
counter-evidence field, overclaim guard, legal/security guardrail, and chapter
role.

No doctrine instrument may be called implementation-ready until it identifies
the legacy artifact it replaces, deletes, narrows, or automates.

No CALM metric may be used without a Metric Integrity Test explaining how the
metric could improve while operational reality worsens.

No learning claim may be promoted from captured to learned without second valid use or an explicitly marked hypothesis.

## Usefulness Check

This pipeline turns "write the book and doctrine" into inspectable gates. It
prevents progress theater by requiring evidence, source maturity, failure
modes, overclaim guards, and Paper CALM reflexivity before stronger claims are
allowed.

## Claim Boundary

This pipeline does not complete Paper Armies, CALM, the source dossier, the
doctrine manual, the toolkit, or empirical validation. It establishes the
sequence and gate criteria for completing them.
