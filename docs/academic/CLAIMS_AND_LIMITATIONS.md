# Claims And Limitations

Status: publication-discipline draft.

## Claims Currently Supportable

1. The repository contains Lean-checked fragments for selected formal
   definitions, implications, blockers, and non-entailment witnesses.
2. The repository contains EFC-bounded finite-check artifacts for 10 selected
   Boolean-kernel targets.
3. The repository contains a PYC proof checker for a small Horn-like
   definitional implication fragment.
4. The repository contains EMP0 empirical protocol artifacts.
5. The repository contains internal comparison and reducibility audit
   artifacts.

## Claims Not Supportable

1. Paralogic is complete.
2. Paralogic is empirically validated.
3. ISFT proves harm, wrongdoing, illegality, guilt, or repair obligation.
4. M8 proves discrimination, coercion, harm, illegality, guilt, or repair.
5. ValidInsight proves empirical truth, moral truth, or repair.
6. EFC-bounded search is global exhaustive search.
7. PYC status is Lean verification.
8. Internal related-work notes are external peer review.

## Publication Wording

Preferred wording:

> Paralogic is a Lean-first formal workbench for studying selected
> non-collapse boundaries among contradiction, institutional support failure,
> evaluator-gated insight, normative bridge warrants, repair stages, and
> empirical validation protocols.

Avoid wording that presents the project as a completed logic, a validated
social-scientific theory, or a legal/moral decision procedure.
